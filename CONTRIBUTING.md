# Contributing to JS to React — 30 Days 🤝

First of all — thanks for taking the time to contribute!
Whether you're fixing a bug, improving an explanation, or suggesting a better approach — all contributions are welcome.

---

## 📌 What You Can Contribute

- Fix incorrect code or typos in any `.js` file
- Suggest a better or cleaner way to explain a concept
- Add an alternative solution to an exercise
- Improve comments for better clarity
- Suggest new exercise questions

---

## 🚀 How to Contribute

### 1. Fork the Repository

Click the **Fork** button on the top right of this repo page.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/js-to-react-30days.git
cd js-to-react-30days
```

### 3. Create a New Branch

Always create a new branch — never work directly on `main`.

```bash
git checkout -b fix/day-01-variable-typo
# or
git checkout -b improve/day-01-loop-explanation
```

Branch naming convention:
- `fix/` — for bug fixes or typo corrections
- `improve/` — for better explanations or cleaner code
- `add/` — for new questions or examples

### 4. Make Your Changes

Edit the relevant file. Keep your changes focused — one fix or improvement per PR.

### 5. Commit Your Changes

```bash
git add .
git commit -m "fix: corrected var hoisting example in day-01"
```

Commit message format:
- `fix:` — bug or typo fix
- `improve:` — better explanation or code
- `add:` — new content

### 6. Push to Your Fork

```bash
git push origin your-branch-name
```

### 7. Raise a Pull Request

- Go to the original repo on GitHub
- Click **Compare & pull request**
- Add a short description of what you changed and why
- Submit the PR

---

## ✅ PR Review Process

- I personally review every PR
- If changes are needed, I'll leave a comment
- Once approved — it gets merged 🎉

---

## 💬 Have a Suggestion?

Open an **Issue** on GitHub with your idea.
Label it as `suggestion` or `question`.

---

## 🙌 Code of Conduct

- Be respectful and constructive
- No gatekeeping — beginners are welcome too
- Focus on learning, not just showing off
